Portfolio Website

Enclosing my share of cyberspace to catalogue my endeavours.

This page offers a brief overview of the decisions I made when choosing the approach and stack (or lack thereof) for this very website.

KISS Won

Ultimately, I opted to construct this site using straightforward HTML, CSS, and frontend JavaScript*

I must say it's a joy to be living in the age of Web Components and the Fetch API. Together they make code reuse painless :❳

*Dirty Secret

TypeScript is nice but I find developing without a build step more fun. I feel like I enjoy JavaScript in the same way some people enjoy free climbing. At least for a project like this one.

Considered alternatives

Wordpress/Other CMS

Reasons For

  • Structure
  • SEO
  • Set-up speed

Reasons Against

  • Would need to code plugins to display certain things in the way I desire
  • Hack-magnet if self-hosted
  • Limitations of a blog-light structure

Neo4j-backed Solution

Reasons For

  • Opportunity to play more with graph databases
  • Flexible sufacing of data and relationships
  • Opportunity to create something more unique/special

Reasons Against

  • Serving content from a database creates new security concerns
  • The stack becomes more complicated
  • No viable CMS means I would have to code my display and query logic
  • Leaning curve presents a project risk; I could waste time building a custom CMS instead of getting my content on the web

Static Site Generator

Reasons For

  • Advantages of dynamic content with static hosting
  • Correct framework

Reasons Against

  • There are multiple to choose from; learning takes time.
  • Build process would become more complex.
  • Attaching external framework too early in this project could be a vector for code rot.
